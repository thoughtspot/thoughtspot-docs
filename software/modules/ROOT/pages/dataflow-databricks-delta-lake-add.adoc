You can add a connection to the Databricks Delta Lake database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the *Connection type*, the rest of the connection properties appear.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-connection-name[Connection name]::
Name your connection.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-connection-type[Connection type]::
Choose the Azure Databricks connection type.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-server-hostname-[Server Hostname]::
Specify the hostname of the Databricks server
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-port[Port]::
Specify the port associated with the system.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-http-path[HTTP path]::
Specify the HTTP Path.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-protocol[Protocol]::
Specify the remote server connection
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-cluster-id[Cluster id]::
Specify the canonical identifier for the cluster
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-access-token[Access token]::
Specify the access token to authenticate Databricks API.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-conn-dbfs-stage-location[DBFS stage location]::
Specify the mount storage object location
+
See xref:dataflow-databricks-delta-lake-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
. Select *Create connection*.
