= Add a Snowflake connection
:last_updated: 02/19/2021
:linkattrs:
:experimental:
:page-aliases: /data-integrate/dataflow/dataflow-snowflake-add.adoc

You can add a connection to a Snowflake database using ThoughtSpot DataFlow.

To add a new connection to Snowflake, follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the Snowflake *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-snowflake-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-connection-type[Connection type]:: Choose the Snowflake connection type.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-user[User]:: Specify the user to connect to Snowflake.
This user must have data access privileges to the databases.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-password[Password]:: Specify the password.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-account[Account]:: Specify the account to connect to the Snowflake database.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-warehouse[Warehouse]:: Choose the warehouse of databases.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-role[Role]:: Specify the role that defines the set of privileges assigned to a user or a user group.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-keep-connection-alive[Keep connection alive]:: Specify if system must force the user to log in again after a period of inactivity.
+
&bull;
When `TRUE`, Snowflake keeps the session active indefinitely, even if there is no user activity. +
&bull;
When set to `FALSE` user must log in after four hours of inactivity.
+

Under *Advanced Options*.
xref:dataflow-snowflake-reference.adoc#dataflow-snowflake-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Click *Create connection*.
