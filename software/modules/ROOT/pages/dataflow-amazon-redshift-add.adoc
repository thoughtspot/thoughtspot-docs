= Add an Amazon Redshift database connection
:last_updated: 02/14/2022
:linkattrs:
:experimental:
:page-aliases: /data-integrate/dataflow/dataflow-amazon-redshift-add.adoc

You can add a connection to an Amazon Redshift database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the Amazon Redshift *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-amazon-redshift-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.

xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-connection-name[Connection name]::  Name your connection.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-connection-type[Connection type]::  Choose the Amazon Redshsift connection type.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-host[Host]::  Specify the hostname or the IP address of the Redshift system
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-port[Port]::  Specify the port associated to the Redshift system
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-user[User]::  Specify the user to connect to Amazon Redshift.
This user must have data access privileges.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-password[Password]::  Specify the password for the User.

xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (e.g AWS Secrets Manager).
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-database[Database]::  Collection of information that is organized so that it can be easily accessed, managed and updated.
xref:dataflow-amazon-redshift-reference.adoc#dataflow-amazon-redshift-conn-jdbc-options[JDBC options]::  Specify the options associated with the JDBC URL.
+

. Click *Create connection*.
