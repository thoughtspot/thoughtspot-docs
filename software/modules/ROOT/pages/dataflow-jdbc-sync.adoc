= Sync data through a JDBC connection
:last_updated: 09/10/2021
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-jdbc-sync.adoc



After using ThoughtSpot DataFlow to establish a connection to a  database through JDBC, you can create automatic data updates, to seamlessly refresh your data.

include::partial$sync-for-databases.adoc[]

. Specify the sync properties for JDBC:
+
See xref:dataflow-jdbc-reference.adoc#sync-properties[Sync properties] for details, defaults, and examples.
+
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-column-delimiter[Column delimiter]:: Specify the column delimiter character.
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-enclosing-character[Enclosing character]:: Specify if the text columns in the source data needs to be enclosed in quotes.
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-escape-character[Escape character]:: Specify this if the text qualifier is mentioned.
This should be the character which escapes the text qualifier character in the source data.
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-fetch size[Fetch size]:: Specify the number of rows to be fetched at a time and processed in memory.
If the value specified is zero then, all rows are extracted at once.
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-max-ignored-rows[Max ignored rows]:: Abort the transaction after encountering 'n' ignored rows.
xref:dataflow-jdbc-reference.adoc#dataflow-jdbc-sync-ts-load-options[TS load options]:: Specifies the parameters passed with the `tsload` command, in addition to the commands already included by the application.
+
The format for these parameters is:
+
[source]
----
--<param_1_name> <optional_param_1_value>
--<param_2_name> <optional_param_2_value>
----
. Save your work by clicking *Save*.
+
Alternatively, click *Save and sync now* to save your work and sync data at the same time.
