= Add an Amazon Aurora database connection
:last_updated: 02/14/2022
:linkattrs:
:experimental:
:page-aliases: /data-integrate/dataflow/dataflow-amazon-aurora-add.adoc
:description: You can add a connection to an Amazon Aurora database using ThoughtSpot DataFlow.



You can add a connection to an Amazon Aurora database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the Amazon Aurora *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-amazon-aurora-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-connection-name[Connection name]::
Name your connection.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-connection-type[Connection type]::
Choose the Amazon Aurora connection type.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-host[Host]::
Specify the hostname or IP address of the Aurora system.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-port[Port]::
Specify the port associated with Amazon Aurora connections.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-user[User]::
Specify the user to connect to Amazon Aurora.
This user must have data access privileges.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-password[Password]::
Specify the password for the User.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (for example, AWS Secrets Manager).
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
xref:dataflow-amazon-aurora-reference.adoc#dataflow-amazon-aurora-conn-jdbc-options[JDBC options]::
Specify the options associated with the JDBC URL.

. Select *Create connection*.
