= Add an Oracle DataFlow connection
:last_updated: 7/6/2020
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-oracle-add.adoc



You can add a connection to an Oracle database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the Oracle *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-oracle-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-connection-type[Connection type]:: Choose the Oracle connection type.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-host[Host]:: Specify the hostname or the IP address of the Oracle system.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-port[Port]:: Specify the port associated to the Oracle system.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-user[User]:: Specify the user id that will be used to connect to the Oracle system.
This user should have necessary privileges to access the data in the databases.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-password[Password]:: Specify the password for the User.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-service-name[Service name]:: This is the TNS alias that you give when you remotely connect to your database.
You can find it in the Tnsnames.ora file.
It can be the same as the SID.
You can also give it any other name you want.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (e.g. AWS Secrets Manager).
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-version[Version]:: Specify the Oracle version being connected to.
xref:dataflow-oracle-reference.adoc#dataflow-oracle-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Click *Create connection*.
