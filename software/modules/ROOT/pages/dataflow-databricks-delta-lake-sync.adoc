After using ThoughtSpot DataFlow to establish a connection to a Databricks Delta Lake file system, you can create automatic data updates, to seamlessly refresh your data.

include::partial$sync-for-databases.adoc[]

. Specify the sync properties for Databricks Delta Lake:
+
See xref:dataflow-databricks-delta-lake-reference.adoc#sync-properties[Sync properties] for details, defaults, and examples.
+
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-data-extraction-mode[Data extraction mode]:: Specify the extraction type.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-column-delimiter[Column delimiter]:: Specify the column delimiter character.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-enclosing-character[Enclosing character]::  Specify if the text columns in the source data needs to be enclosed in quotes.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-escape-character[Escape character]::  Specify this if the text qualifier is mentioned.
This should be the character which escapes the text qualifier character in the source data.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-max-ignored-rows[Max ignored rows]:: Abort the transaction after encountering 'n' ignored rows.
xref:dataflow-databricks-delta-lake-reference.adoc#dataflow-databricks-delta-lake-sync-ts-load-options[TS load options]::  Specifies the parameters passed with the `tsload` command, in addition to the commands already included by the application.
+
The format for these parameters is:
+
[source]
----
--<param_1_name> <optional_param_1_value>
--<param_2_name> <optional_param_2_value>
----
. Save your work by clicking *Save*.
+
Alternatively, click *Save and sync now* to save your work and sync data at the same time.
