= Add a Netezza database connection
:last_updated: 2/14/2022
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-netezza-add.adoc
:description: You can add a connection to a Netezza database using ThoughtSpot DataFlow.



You can add a connection to a Netezza database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the Netezza *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-netezza-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-connection-type[Connection type]:: Choose the Netezza connection type.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-host[Host]:: Specify the hostname or the IP address of the Netezza system.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-port[Port]:: Specify the port associated to the Netezza system.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-user[User]:: Specify the user id that will be used to connect to the Netezza system.
This user should have necessary privileges to access the data in the databases.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-password[Password]:: Specify the password for the User.
+
////
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-version[Version]:: Specify the Netezza version being connected to.
////
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (e.g. AWS Secrets Manager).
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
xref:dataflow-netezza-reference.adoc#dataflow-netezza-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Click *Create connection*.
