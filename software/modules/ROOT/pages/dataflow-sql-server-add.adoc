= Add an SQL Server DataFlow connection
:last_updated: 02/19/2021
:linkattrs:
:experimental:
:page-aliases: /data-integrate/dataflow/dataflow-sql-server-add.adoc

You can add a connection to an SQL Server database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the SQL Server *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-sql-server-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-connection-type[Connection type]:: Choose the SQL Server connection type.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-host[Host]:: Specify the hostname or the IP address of the SQL Server system.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-port[Port]:: Specify the port associated with the SQL Server system.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-user[User]:: Specify the user id that will be used to connect to the SQL Server system.
This user should have necessary privileges to access the data in the databases.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-password[Password]:: Specify the password for the User.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-instance[Instance]:: Specify an instance with all SQL Server databases run by a single SQL Server service
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-version[Version]:: Specify the SQL Server version.
xref:dataflow-sql-server-reference.adoc#dataflow-sql-server-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Click *Create connection*.
