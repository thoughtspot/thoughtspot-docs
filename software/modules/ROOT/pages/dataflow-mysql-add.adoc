= Add a MySQL database connection
:last_updated: 2/14/2022
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-mysql-add.adoc



You can add a connection to a MySQL database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the MySQL *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-mysql-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-connection-type[Connection type]:: Choose the MySQL connection type.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-host[Host]:: Specify the hostname or the IP address of the MySQL system.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-port[Port]:: Specify the port associated to the MySQL system.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-user[User]:: Specify the user id that will be used to connect to the MySQL system.
This user should have necessary privileges to access the data in the databases
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-password[Password]:: Specify the password for the App User.
+
////
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-version[Version]:: Specify the MYSQL version being connected to.
////
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (for example, AWS Secrets Manager).
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
xref:dataflow-mysql-reference.adoc#dataflow-mysql-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Select *Create connection*.

'''
