= Sync data through a Vertica connection
:last_updated: 9/10/2021
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-vertica-sync.adoc

After using ThoughtSpot DataFlow to establish a connection to a Vertica database, you can create automatic data updates, to seamlessly refresh your data.

include::partial$sync-for-databases.adoc[]

. Specify the sync properties for Vertica:
+
See xref:dataflow-vertica-reference.adoc#sync-properties[Sync properties] for details, defaults, and examples.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-column-delimiter[Column delimiter]:: Specify the column delimiter character.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-escape-character[Escape character]:: Specify the escape character if using a text qualifier in the source data.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-enclosing-character[Enclosing character]:: Specify if the text columns in the source data must be in quotes, and the quote types.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-ts-load-options[TS load options]:: Specifies the parameters passed with the `tsload` command, in addition to the commands already included by the application.
+
The format for these parameters is:
+
----
--<param_1_name><optonal_param_1_value>
--<param_2_name><optonal_param_2_value>
----

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-max-ignored-rows[Max ignored rows]:: Terminate the transaction when encounter `n` ignored rows.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-fetch-size[Fetch size]:: Specify the number of rows to fetch at one time, and process in memory.
+
If the value specified is `0`, then DataFlow extracts all rows at the same time.

xref:dataflow-vertica-reference.adoc#dataflow-vertica-sync-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Save your work by clicking *Save*.
+
Alternatively, click *Save and sync now* to save your work and sync data at the same time.
