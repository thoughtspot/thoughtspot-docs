= Sync data through an Impala connection
:last_updated: 7/03/2020
:experimental:
:linkattrs:
:page-aliases:

After using ThoughtSpot DataFlow to establish a connection to an Impala database, you can create automatic data updates, to seamlessly refresh your data.

include::partial$sync-for-databases.adoc[]

. Specify the sync properties for Impala:
+
See xref:dataflow-impala-reference.adoc#sync-properties[Sync properties] for details, defaults, and examples.
+
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-data-extraction-mode[Data extraction mode]:: Specify the extraction type.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-enclosing-character[Enclosing character]:: Specify if the text columns in the source data needs to be enclosed in quotes.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-escape-character[Escape character]:: Specify this if the text qualifier is mentioned. This should be the character which escapes the text qualifier character in the source data.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-fetch-size[Fetch size]:: Specify the number of rows to be fetched at a time and processed in memory.
If the value specified is zero then, all rows are extracted at once.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-max-ignored-rows[Max ignored rows]:: Abort the transaction after encountering 'n' ignored rows.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-column-delimiter[Column delimiter]:: Specify the column delimiter character.
xref:dataflow-impala-reference.adoc#dataflow-impala-sync-ts-load-options[TS load options]:: Specifies the parameters passed with the `tsload` command, in addition to the commands already included by the application.
+
The format for these parameters is:
+
----
--<param_1_name><optonal_param_1_value>
--<param_2_name><optonal_param_21_value>
----


. Save your work by clicking *Save*.
+
Alternatively, click *Save and sync now* to save your work and sync data at the same time.
