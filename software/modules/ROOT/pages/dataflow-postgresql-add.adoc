= Add a PostgreSQL database connection
:last_updated: 2/14/2022
:experimental:
:linkattrs:
:page-aliases: /data-integrate/dataflow/dataflow-postgresql-add.adoc
:description: You can add a connection to a PostgreSQL database using ThoughtSpot DataFlow.



You can add a connection to a PostgreSQL database using ThoughtSpot DataFlow.

Follow these steps:

include::partial$add-database-connection.adoc[]

. After you select the PostgreSQL *Connection type*, the rest of the connection properties appear.
+
See xref:dataflow-postgresql-reference.adoc#connection-properties[Connection properties] for details, defaults, and examples.
+
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-connection-name[Connection name]:: Name your connection.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-connection-type[Connection type]:: Choose the PostgreSQL connection type.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-host[Host]:: Specify the hostname or the IP address of the PostgreSQL system.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-port[Port]:: Specify the port associated to the PostgreSQL system.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-database[Database]:: Collection of information that is organized so that it can be easily accessed, managed and updated.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-user[User]:: Specify the user id that will be used to connect to the PostgreSQL system.
This user should have necessary privileges to access the data in the databases.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-password[Password]:: Specify the password for the User.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-fetch-password-from-vault[Enter password/Fetch password from vault]:: Select whether to enter password or fetch and use the password you have stored in external vault providers (e.g. AWS Secrets Manager).
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-vault-provider[Vault provider]:: Select the name of your external vault provider.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-secrets-manager-region[Secrets manager region]:: Region of the AWS account where secret manager is defined.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-secret-name-or-arn[Secret name or ARN]:: If user trying to access using two different accounts, then the value to be given here is a fully qualified Amazon Resource Name (ARN) of the secret.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-remote-role-arn[Remote role ARN]:: If user trying to access using two different accounts, then the value to be given here is a remote role Amazon Resource Name (ARN) where secret manager is not defined.
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-key-name-in-secret[Key name in secret]:: Key name given for the secret, to get its value.
+
////
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-version[Version]:: Specify the PostgreSQL version being connected to.
////
xref:dataflow-postgresql-reference.adoc#dataflow-postgresql-conn-jdbc-options[JDBC options]:: Specify the options associated with the JDBC URL.

. Select *Create connection*.

'''
