= Orgsを使用したマルチテナンシー
:last_updated: 2022年9月23日
:linkattrs:
:experimental:
:page-layout: default-cloud
:description: ThoughtSpot のマルチテナント機能について理解しましょう。この機能により、インスタンスを論理的に複数の環境に分割することができます。

マルチテナント機能は、ThoughtSpot クラウド インスタンスを、*Orgs*と呼ばれる複数のテナント固有の環境に論理的に分割します。Orgsを使用すると、各テナントのデータはアクセス制御により分離され、保護され、同じThoughtSpotアプリケーションインスタンスを共有する他のテナントからは見えません。

インスタンスで Orgs 機能が有効になっている場合、クラスター管理者はテナントアカウントごとに Org を作成し、グループとユーザーを構成し、データオブジェクトへのアクセスを制御できます。各組織は、それぞれのユーザーとデータのセットを持つ独立したコンテナとして機能し、通常のThoughtSpotインスタンスと同じユーザーエクスペリエンスを提供します。

[NOTE]
====
Orgs 機能には、ThoughtSpot Enterprise Edition または ThoughtSpot Embedded Enterprise Edition が必要です。https://www.thoughtspot.com/pricing[ThoughtSpot の価格とエディション^]をご覧ください。

これらのエディションのいずれかをお持ちの場合は、{support-url}に連絡してOrgsを有効にしてください。Orgs は、すべての新しいエンタープライズ顧客に対してデフォルトで有効になっています。今後のリリースでは、既存の顧客クラスターに対して組織が有効になる予定です。組織の代わりにグループを使用したマルチテナントの詳細については、https://developers.thoughtspot.com/docs/multitenancy-without-orgs[グループによるマルチテナント^]を参照してください。

もし新規のThoughtSpot顧客で、クラスターでOrgsが有効になっていない場合は、{support-url}にご連絡ください。

一度有効にした組織は無効にできないことにご注意ください。 組織を有効にするために必要な変更については、以下をご覧ください。
====

この記事は、標準の ThoughtSpot Cloud 環境におけるオーガニゼーション機能とオーガニゼーション管理の概要です。ThoughtSpot開発者環境でのREST APIを使用した組織の機能とプログラムによる組織管理の詳細については、https://developers.thoughtspot.com/docs/?pageid=orgs[組織によるマルチテナント^]をご参照ください。組織APIの詳細については、https://developers.thoughtspot.com/docs/?pageid=orgs-api[組織API<4>}をご参照ください。^]

NOTE: クラスターで組織機能を有効にした後は、組織を作成する機能を無効にすることはできません。ただし、組織を作成するまで、環境は単一テナント環境のままです。組織が有効になっている単一テナント環境が必要な場合は、作成したすべての組織を削除し、プライマリ組織のみを使用することもできます。詳細については、<<enable-disable,「組織の有効化と無効化」を>>ご参照ください。

== ThoughtSpotの組織

インスタンスで Orgs 機能が有効になっている場合、デフォルトでは 1 つの Org のみが存在します。ThoughtSpot では、このデフォルトの組織を *プライマリ組織*と呼びます。*クラスター管理者* はプライマリ組織の管理者です。プライマリ組織の管理者は、さらに組織を作成し、各組織に管理者ユーザープロファイルを追加してユーザーとグループを管理し、組織レベルでアクセス制御を定義できます。他の組織の管理者 ( *組織管理者*と呼ばれます) は、その組織のみを管理できます。プライマリ組織の管理者のみが全ての組織を管理できます。

次の図は、マルチテナント環境におけるThoughtSpotインスタンス上のOrgsアーキテクチャを示しています。

image::org-logical-architecture.png[組織の論理アーキテクチャ図]

[NOTE]
====
* プライマリ組織は削除できません。
* 各組織は一意の名前で識別され、独自のユーザーとグループを持つことができます。
* グループは組織に固有であり、組織のコンテキスト内でのみ作成できます。
* ユーザーは複数の組織に所属できます。
====

== マルチテナントの分離

ThoughtSpot は、マルチテナント環境における組織の完全な分離を保証します。具体的には、Orgs 機能を使用する場合、ThoughtSpot は次のすべてを特定の Org に制限します。

* クラウドデータウェアハウスの接続
* クラウドデータウェアハウスデータ
* グループ
* ユーザー
** ユーザーは複数の組織に所属できますが、一度にアクセスできるのは現在の組織のデータとオブジェクトのみであることにご注意ください。
** ユーザーが複数の組織に属している場合は、現在の組織コンテキストを所属する他の組織のいずれかに切り替えることができます。
** ユーザーは自分が所属していない組織について認識することはありません。
* ThoughtSpot オブジェクト:ライブボード、ワークシート、アンサー、テーブル。組織間でオブジェクトを共有することはできません。
* データコンテンツの検索とインデックス
* アンサーのコンテンツとインデックスの検索
* その他のインデックス

== ユーザーロール

マルチテナントインスタンスでは、ThoughtSpot は次の 3 つのユーザーロールをサポートします：クラスター管理者、組織管理者、組織ユーザー。デフォルトでは、プライマリ組織の管理者はクラスタ管理者です。クラスター管理者はすべての組織にアクセスし、すべての組織に対して CRUD (作成、読み取り、更新、削除) 操作を実行できます。クラスター管理者は、組織間を切り替えたり、組織管理者としても機能できます。

組織管理者は、それぞれの組織のユーザーとグループのみを管理でき、すべての組織を管理することはできません。ユーザーは複数の組織に所属でき、所属するグループに割り当てられた権限に基づいてデータオブジェクトにアクセスできます。次の図は、マルチテナントのThoughtSpotインスタンス上でのユーザー設定を示しています。

image::org-user-setup-diagram.png[組織ユーザーの設定図]

=== クラスタ管理者と組織管理者

クラスター管理者と組織管理者には、同じ権限と責任がいくつかあります。これらの役割は重複していますが、同じではありません。

*クラスター管理者*は次のタスクを実行できます：

* 組織管理：組織の作成、編集、削除
* あらゆる組織のあらゆるユーザーに対するユーザー管理
* あらゆる組織内のあらゆるグループの管理
* 接続管理：任意の組織内の任意の接続の作成、編集、削除クラスター管理者は、*データを管理できる*権限を持つ組織管理者やユーザーとxref:connections.adoc#connection-share[接続を共有]することもできます。これにより、これらのユーザーは接続内のテーブルを追加、削除、変更することができます。

クラスター管理者の場合、*プライマリ組織* の管理コンソールには、組織間の構成を管理するための *すべての組織* セクションと、プライマリ組織の構成を管理するための *プライマリ組織* セクションがあります。

image::org-cluster-admin.png[クラスタ管理者向けの組織管理コンソール]

*組織管理者*は次のタスクを実行できます。

* 組織管理者が管理者権限を持つ組織内のすべてのユーザーのユーザー管理を行います。
* 組織管理者が管理者権限を持つ組織内の任意のグループの管理を行います。
* 接続管理: 組織管理者が管理者権限を持つ組織内の接続を作成、編集、削除します。デフォルトでは、クラスター管理者が接続を組織管理者と共有しない限り、組織管理者にはクラスター管理者が作成した接続を表示または編集するアクセス権がありません。クラスター管理者が xref:connections.adoc#connection-share[接続を共有する] と、組織管理者はその接続内のテーブルを追加したり、削除したり、変更したりできます。
* *_[TSE専用]_*https://developers.thoughtspot.com/docs/?pageid=trusted-auth[信頼性のある認証設定^]:ThoughtSpot Embedded ユーザーは、信頼性のある認証を設定することができます。これにより、ユーザーが ThoughtSpot に直接ログインしたり、第三者の IdP にリダイレクトされることなく、ブラウザで ThoughtSpot のセッションを開始することができます。
+
NOTE: 組織専用の信頼性のある認証と組織専用の *開発* タブを有効にするためには、{support-url}にお問い合わせください。

組織管理者の場合、管理コンソールには *[すべての組織]* セクションは *_存在しません_* 。組織管理者は、自身の特定の組織の設定のみを管理することができます。

image::org-org-admin.png[組織管理者向けの組織管理コンソール]

== 組織を作成し、管理します

マルチテナントのThoughtSpotインスタンス上で組織を作成するには、クラスター管理者の権限が必要です。デフォルト設定では、プライマリ組織の管理者はクラスター管理者となります。クラスター管理者は、UIまたはREST APIエンドポイントを使用して組織を作成することができます。クラスター管理者は、組織の編集や削除も行うことができます。

組織の作成、編集、削除の方法については、xref:orgs-administration-overview.adoc#create-manage[組織の管理]をご参照ください。

REST APIエンドポイントを使用して組織を作成する方法については、https://developers.thoughtspot.com/docs/?pageid=orgs-api[組織 API^]をご参照ください。

[#switch]
== 組織間を切り替える

複数の組織にアクセス可能なユーザーは、ヘルプアイコンの隣にある上部ナビゲーションバーの*組織スイッチャー*を使用して、所属する組織間を切り替えることができます。*Org スイッチャー* は、ユーザーが現在所属している Org も識別します。たとえば、次の画像では、ユーザーはプライマリ組織に所属しています。

image::org-switcher.png[Org スイッチャー]

ユーザーは、Org スイッチャーで自分が所属する組織のみを表示できます。ユーザーが 1 つの Org のみに所属している場合、Org 間を切り替えるオプションはありません。表示されるのは、自分が所属する 1 つの組織のみです。

クラスター管理者は、複数の Org オブジェクトにアクセスし、構成の変更を適用できます。Org 内のオブジェクトに対して何らかの操作を実行する前に、管理者は正しい Org を指定して、関連する Org のコンテキストに構成の変更が適用されるようにする必要があります。

image::org-switcher-gif.gif[Org を切り替えます]



== 全組織の範囲

マルチテナント型のThoughtSpotインスタンスでは、全ての操作はユーザーが現在所属している組織のコンテキストで行われます。クラスター管理者が CRUD (作成、読み取り、更新、または削除) 操作を行ったり、全組織に設定変更を適用したりする場合、管理コンソールの *全組織* セクションにてプライマリ組織にいる必要があります。

例えば、複数の組織にユーザーを追加するには、管理者は管理コンソールの *[全組織] > [ユーザー]* セクションにてプライマリ組織にいる必要があります。例えば、管理者が *Org 1* にいる場合、管理コンソールの *ユーザー* セクションには、ユーザーを組織に追加するオプションはなく、グループに追加するオプションのみがあります。複数の組織にユーザーを追加する場合は、管理コンソールの *すべての組織 > ユーザー* セクションのみをご利用ください。個々の組織にユーザーを追加したり、グループにユーザーを追加したりする場合は、各組織の *管理コンソール > ユーザー* セクションをご利用ください。

== ユーザーとグループの管理を行います

マルチテナントインスタンスでは、クラスター管理者と組織管理者はユーザーとグループを作成し、管理することができます。クラスター管理者は、全ての組織のユーザーおよびグループオブジェクトに対してCRUD（作成、読み取り、更新、削除）操作を行うことができます。一方、組織管理者は自身が所属する組織のコンテキスト内でのみユーザーとグループを作成し、管理することができます。

[NOTE]
====
Org グループを管理するためには、クラスタ管理者は適切な Org に切り替える必要があります。これには、特定のユーザーが属するグループの管理が含まれます。Org ユーザーを管理するには、クラスター管理者はプライマリ Org またはユーザーが属する特定の Org にいることができます。

複数の組織にユーザーを追加する場合は、管理コンソールの *すべての組織 > ユーザー* セクションのみを使用してください。個々の組織の *管理コンソール > ユーザー* セクションを使用して、個々の組織にユーザーを追加したり、グループにユーザーを追加したりできます。
====

Orgs内のユーザーとグループを追加、削除、管理する方法については、xref:orgs-administration-overview.adoc#users-groups[Orgsの管理]を参照してください。

== 組織の有効化と無効化
Orgs 機能には、ThoughtSpot Enterprise Edition または ThoughtSpot Embedded Enterprise Edition が必要です。https://www.thoughtspot.com/pricing[ThoughtSpot の価格とエディション^]をご覧ください。すべての新しいクラスターでは、Org がデフォルトで有効になっています。お使いの環境でOrgsを有効にするには、{support-url}にご連絡ください。

クラスターで Orgs 機能を有効にした後は、Orgs を作成する機能を無効にすることはできません。ただし、Orgを作成するまでは、環境は単一テナント環境のままです。Orgsが有効になっている単一テナント環境が必要な場合は、作成したすべてのOrgsを削除し、プライマリOrgのみを使用することもできます。

Org を有効にした後、マルチテナントを利用しないことにした場合、ThoughtSpot 環境は次のように変わります。

* あなたの管理コンソールは、標準の管理コンソールの代わりにxref:admin-console-orgs.adoc[]のように見えるでしょう。Orgs 管理コンソールには、標準の管理コンソールと同じ機能がすべて備わっています。Org を作成する機能もあります。
* 上部のナビゲーションバーには、ヘルプアイコンの横に<<switch,組織スイッチャー>>が表示されます。しかし、もし何もOrgを作成しない場合、Orgスイッチャーは何も機能しません。
